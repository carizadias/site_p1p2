Project 2 Description

"One More Cup of Coffee" is a project centered around creating a room-scale Virtual Reality environment. This project offers a unique opportunity for developers to explore the construction of virtual worlds where individuals can interact and move, experiencing this reality at a human scale.

Inspired by the concept of a familiar kiosk, it harks back to memories of days spent selling donuts and hot chocolate in the atrium of a university building. The aim is to simulate the experience of a similar kiosk within the new Computer Science building even before its completion.

Starting with an initial kit as a foundation, the project expands by adding models, sounds, lighting, and interactions to enrich this virtual space. The environment is geared towards physical exploration, inviting users to walk and engage with the space through natural gestures, utilizing Oculus Quest headsets.

The central idea is not only to create a visually immersive space but also a functional one. This involves incorporating various models, from static objects to interactive items that respond to user gestures. Additionally, including characters and seating areas for customers adds layers of realism and social interaction to the environment.

The project aims not just to create an immersive environment but also to provide a platform for experimentation and learning, utilizing technologies such as Unity 3D and VRTK (Virtual Reality Toolkit) version 4. This is an invitation to explore the possibilities of Virtual Reality while developing a virtually engaging and functional space for users.

Unity Project setup

The "One More Cup of Coffee" project kicks off by opening Unity Hub and ensuring Unity Editor 2021.3.6f1 is installed. Instead of starting a new project, we dive into the provided starter project from the assignment. This endeavor revolves around crafting a room-scale VR environment, inspired by memories of vending in a university atrium.

Beginning with a foundational kit, we expand it with bespoke models, sounds, lighting, and interactive elements. This enhances the space, inviting users to explore using Oculus Quest headsets, engaging through natural gestures.

Our aim is not just visual allure but functionality. We blend static and responsive elements, integrating animated characters and seating for an immersive social touch.

This project isn't just about creating; it's a learning platform, utilizing Unity 3D and VRTK (Virtual Reality Toolkit) version 4. It's an invitation to explore VR possibilities while building an engaging, functional digital space.

My kiosk Modeling

The starter project already contains the building and several other models that I could utilize. My concept revolves around creating a simple wooden plant store. The starter package includes essential elements like a freezer, various cabinets, and numerous plant models. Additionally, I found extra plant models from trustworthy sources like Free3D, TurboSquid, and Sketchfab, which offer a wide array of downloadable models.

Combining these assets forms the core of my envisioned wooden plant shop. The deliberate arrangement within the scene blends wooden aesthetics with diverse plant species, creating a serene and inviting atmosphere.

To bring this digital canvas to life, I made precise lighting adjustments. These modifications not only enhance the visual appeal but also guide users through this digital sanctuary.

Make a human

This part posed the most significant challenge for me as I had never attempted MakeHuman before. It became an entirely new experience. To begin, I navigated to the official MakeHuman website at MakeHuman Community. In the Downloads section, I selected the recommended version compatible with my operating system, Windows 10. Please note that installation procedures might differ for various specificities. I'll detail the installation process for Windows 10, although the process may be similar across platforms.

Upon downloading, I located the downloaded file and executed the .exe file to initiate the installation process. I ensured that the checkbox for creating a desktop shortcut was enabled during the installation. Subsequently, a shortcut icon appeared on my desktop, facilitating quick access to MakeHuman.

Launching MakeHuman introduced a plethora of options for creating customized human models. I found this video particularly helpful for detailed guidance. However, for this particular project, certain specifications are essential. For instance, while exporting the project, it's crucial to save the file as an .fbx format for seamless integration with Unity. Additionally, ensuring that the eyes remain low-poly is vital to prevent potential issues where the eyes might appear blue without proper materials.

Following these steps, I simply dragged the .fbx file into the assets folder and then into the scene/hierarchy within Unity. The downloaded file comes with a 'textures' folder containing textures for your model. Dragging these textures into the assets folder ensures that the human model appears in Unity complete with all the materials used in its creation.

Mixamo

Next, the goal is to breathe life into this character using Mixamo (Mixamo). If you don't have an account, signing up is necessary. With Mixamo, you can select characters or download character movements. However, for this project, we'll directly upload the character created earlier in MakeHuman.

Begin by clicking the "Upload Character" button on Mixamo and drag and drop your .fbx file. If prompted to place movement circles on the character, it likely pertains to the pose chosen in MakeHuman. Opting for the "T" pose helps Mixamo identify the character's skeleton automatically.

After choosing your animation and downloading it, you'll be prompted to select the number of frames (optimal at 60 for smoother transitions). Opt for skin attachment for ease. Drag the downloaded file into the Unity assets folder, replacing the previous character without motion.

However, upon adding the player model, you'll notice that the character isn't moving yet. There are a few more crucial steps. Select your character and navigate to the Inspector tab. Add the Animator component (if not visible initially, type it into the search bar).

The missing component here is the Animator Controller, crucial for enabling movement. Right-click in the assets window, select "Create," and generate an Animator Controller. To the right of the Scene tab, the Animator tab will appear. Click on the Animator tab, where you'll already be provided with the Entry, Any State, and Exit states. Above the Entry state, create a new state by right-clicking and selecting 'Create State' or 'New Blend Tree.' Name this state, then right-click the Entry state and create a transition to the newly created state. Return to the Inspector tab, click the circle symbol in the controller field, and select your controller. Switch to the player mode, and you'll witness your character's animation coming alive within your scene.

Deployment

As you might have observed, this project is considerably weightier due to its numerous files. Apart from the ones you've created, it encompasses the entire building from the provided project. However, the GitHub setup is akin to Project P1 (feel free to take a look). Yet, this time, due to its bulkier nature compared to P1, I've committed the changes in stages, adding one file at a time. I'm aware that this might not be the most ethical or professional approach, but it proved to be the only method that functioned seamlessly. The weight and complexity of the project demanded a different commit strategy to ensure the repository's smooth functioning while maintaining a comprehensive track of changes made throughout the development process.
