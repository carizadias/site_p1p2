Project 1 Description

This project was undertaken in four key steps:

First the integration of the Vuforia Engine;

Second, the intricate modeling of landscapes, including the 'Poilão' tree in the 'Boa Entrada' area in Santiago, Cabo Verde, and the 'Santa Monica' beach in 'Boa Vista,' also in Cabo Verde;

Third, the implementation of the OpenWeather API to display location-based information, current date/time, and temperature/climate data;

And finally, the deployment process tailored for Android mobile devices.

Unity Project setup

My project kicked off on Windows, starting with downloading the Unity Hub. It usually prompts a licensing choice among three options. I usually navigate to 'Manage Licences' and pick the last one, 'Personal.' I used Unity Editor version 2023.3.0a10. Initially, any version from 2022 onwards sufficed for this project.

Creating a basic 3D Unity project involves installing the desired editor. This setup is a smart move, helping catch potential bugs early on. Launching the editor and adjusting 'Player Settings' in 'Build Settings' is key. Under 'Player' > 'Other Settings,' set 'Scripting Backend' to 'IL2CPP' and check 'ARM64' under 'Target Architectures.' Devices with ARMv7 might clash with Vuforia, so be careful here.

Set 'Minimum API Level' to 30 or above to avoid errors linked to Google dropping support for ARMv7 APIs. Keep 'Target API Level' as 'Automatic (Highest Installed)' to prevent compatibility issues.

I faced a Vulkan error in 'Graphics APIs.' Solving this required removing Vulkan, leaving only 'OpenGLES3' selected, as shown in the illustration.

In 'XR Plug-in Management,' ticking 'Ignore Gradle Version' under 'Google ARCore' is a good idea if prompted for a download.

For deployment on GitHub, confirm 'Asset Serialization' is 'Force Text' in the 'Editor' section. Also, check 'Reduce Version Control Noise.'

These settings provide a base, but more detailed project configurations will be on the GitHub link for better understanding.

Vuforia Engine

The Vuforia Engine is a potent augmented reality (AR) development platform known for its smooth integration and robust capabilities. Equipped with a diverse set of tools and APIs, Vuforia simplifies crafting immersive AR experiences across different sectors. Thanks to its advanced computer vision technology, developers can effortlessly incorporate both marker-based and markerless AR, enabling apps to recognize and interact with real-world objects. Its adaptability makes it a pivotal choice for AR applications, spanning gaming, education, retail, and industrial solutions.

To integrate Vuforia Engine into my project, I first created an account on their website at Vuforia. Under the 'Downloads' tab, I selected 'Add Vuforia to Unity Project,' initiating a specific file download. Once downloaded, I opened Unity and dragged this file into the 'Assets' folder, then clicked 'Import.'

In the Unity project, I removed the default 'Main Camera' and added an 'AR Camera' by selecting the 'Vuforia Engine' option with the '+' symbol. Organizing the AR Camera as the primary element in the hierarchy helped prevent potential future issues.

Inspecting the AR Camera via the Inspector window, I located the 'Open Vuforia Engine Configuration' button, prompting a request for a license key. By clicking 'Add License,' I accessed the Vuforia webpage. In the 'Develop' section, I entered the 'License Manager.'

Selecting 'Get Basic' in the License Manager allowed me to name the license key and confirm details, revealing the unique key. Copying this key and pasting it into the 'App License Key' field within Unity completed the seamless integration of the Vuforia Engine into my project.

Vuforia multitargets

Incorporating multitargets is a pivotal but complex phase that significantly impacts project outcomes. A lack of precise measurements led to mismatched proportions between the targets, causing a critical issue: Vuforia's inability to recognize both models together due to their varying sizes. This problem persisted, demanding resolution.

Creating multitargets involves revisiting the Vuforia webpage's 'Develop' section and accessing the 'Target Manager.' Here, starting a new database involves clicking 'Add Database,' naming it, and choosing 'Device.' The subsequent steps include defining the target type, providing measurements, naming it, and clicking 'Add.' Then, a detailed configuration follows, uploading and positioning various parts accurately using the 'Merge' and 'Cubes' classes from project resources.

Once completed, downloading the configured database is essential. Like the Vuforia import process, integrating this database into the Unity project requires importing the file into the 'Assets' directory. Within the Unity project hierarchy, creating a multitarget involves right-clicking, selecting 'Vuforia Engine,' and picking 'Multitarget.' Associating this multitarget with the specific database involves repeating this process for the second cube, ensuring its configuration matches the initial setup.

If the multitarget doesn't appear in the scene, double-clicking within the project hierarchy allows visibility and adjustments. This thorough process ensures accurate multitarget configuration, critical for optimal functionality within the augmented reality space.

Modeling

The modeling phase was a creatively liberating part of the project, offering artistic freedom. Initially, we downloaded three models directly into our scene and created at least two more models in Unity. Before that, I made an initial step by creating a 3D object (a cube) that enclosed the multitarget. This cube became the focal point for content display when aligned with the camera.

This cube represented a small version containing landscapes. To achieve this, I added textures to these cubes. Creating a new material involved right-clicking in the Assets section, selecting 'Create/Material,' importing the desired image into the Assets folder, and assigning it as the texture by dragging it into the 'Albedo' field within the material properties. Later, I applied this material to the 3D object by navigating to 'Materials,' clicking the circular symbol, and choosing the newly created material.

I gathered various 3D models from free sources like Free3D and Sketchfab's Free 3D Models. For the 'Class Cube,' models such as a fence, stone, and bush were used. For the 'Merge Cube,' I imported default settings with beach chairs, rocks, sand mounds, and a bucket. In Unity, I focused on basic shapes (cubes, spheres, capsules) and avoided complex scripting.

This creative process aimed to enhance the project, using primary shapes and basic techniques for visual cohesion and a functional augmented reality experience.

OpenWeather API

Integrating the OpenWeather API brought about initial challenges, yet eventually unfolded as an intriguing albeit slightly demanding phase. To begin, having an account on the OpenWeather platform is a prerequisite. After account creation, accessing the dashboard is crucial. Subscribing to the free plan is the next step. Inside the account settings, accessing the 'API Keys' section is pivotal. If a key isn't available, generating one by clicking 'Generate' is essential.

Different plans offer distinct API calls, enabling customization based on information and location specifics. Despite navigating complexities on the OpenWeather website, the basic structure of the free plan's API call typically resembles 'https://api.openweathermap.org/data/2.5/weather?...'—where the ellipsis denotes specific information parameters, followed by the API key.

Insights into the API's functionality and data retrieval process reside in the 'OpenWeatherAPI.cs' file in the project repository. This file contains the API call and logical implementations for retrieving location, weather, and date/time data for each cube's location. In Unity, creating a new script in the Assets folder, naming it meticulously, and dragging it onto the Inspector panel above the Vuforia behavior file (found in the AR camera's Inspector) is essential.

Implementing logic within the script should populate resultant fields. Within Unity, adding a 'UI TextMeshPro' to the cube is crucial. After configuring this UI element, dragging it into the corresponding field beneath the 'OpenWeatherAPI.cs' script is necessary.

Testing the project in 'Play' mode verifies functionality. Webcam-related issues might occur during this phase. I encountered and resolved this issue by transitioning from Ubuntu 22.04 to Windows, reinstalling Unity, and restarting the project—effectively solving the webcam problem during 'Play' mode.

Deployment

Prioritizing a comprehensive approach, I suggest initiating the 'Build and Run' phase before delving into modeling intricacies and OpenWeather API integration. Post Vuforia setup and the establishment of targets, especially when arranging cubes for multitargets, conducting a 'Build and Run' procedure offers substantial insight into real-time project functionality.

Start by establishing a connection between your mobile device and the PC—a critical step. Utilize the original mobile device cable for optimal connectivity. On your mobile device, navigate to settings, access 'About Phone,' and locate 'Build Number.' Tap this number seven times to reveal 'Developer Options.' Within this menu, go to 'USB Debugging' and activate it. Consider enabling 'Stay Awake' to prevent screen locking during application execution. Activate 'Install via USB,' 'USB Debugging (Security Settings),' 'Disable USB Debugging Timeouts,' and 'Force Allow Apps on External' for smooth application execution.

During this setup, encountering bugs, particularly concerning device permissions for accepting unknown files, is common. Assign permissions to relevant applications, especially those associated with project downloads like Google, to preemptively resolve potential permission-related issues.

Transitioning to Unity, navigate to 'Build Settings' and select 'Run Device.' Find your device; if it doesn't appear immediately, manually input its IP address or click 'Refresh.' Initial attempts may not yield immediate results, so several iterations might be necessary. Ensure 'Add Open Scenes' is selected, check 'Development Build,' and finally, execute 'Build and Run.' The loading phase may take time, but the application will automatically launch on your mobile device upon completion.

The application prompts for camera permissions and should seamlessly function, leveraging the device's camera capabilities for augmented reality experiences. Note: Avoid using a Redmi Note phone as it may show switched colors in the background (red turns blue, etc.). Deploying on a Samsung 14A worked without issues, although the cause of this discrepancy is unknown.
